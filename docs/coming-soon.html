What’s next for Felix

Everything on this page is in development. It builds on the indexing, rules, notes, tasks, and workflows already shipping in this repo.

Documentation crawler

A companion service that mirrors the code indexer—this time for external docs.

  • Crawl documentation sites, collapse them into structured components, and reuse Felix search + embeddings.
  • Link doc snippets directly to code components so the AI sees both views in one context bundle.
  • Expose the cached docs inside the same Explore workspace.

Rule-aware chat UI

A lightweight front-end for Claude Code / Codex / LangGraph sessions with Felix baked in.

  • Shows rule bundles before every message so you see exactly what’s injected.
  • Quick toggles to enable/disable rules, re-run validation, or open the linked context.
  • Built on the existing MCP integration scripts in integrations/.

Composable workspace

Drag-and-drop panels for AI chat, code explorer, terminals, and Felix widgets.

  • Arrange widgets (Explore, Component Map, Tasks, Notes, Chat) into custom layouts.
  • All panes communicate over MCP so the AI, Felix, and the IDE stay in sync.
  • Aim: deliver a full IDE experience built around the Felix data layer.
